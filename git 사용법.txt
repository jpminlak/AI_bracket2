0-1. git 업로드용 폴더를 따로 하나 만든다. 폴더를 비운 상태에서 껍데기만 남긴다.

0-2. 작업 시작 직전에 팀원들에게 디스코드(또는 카카오톡 등의 메신저)를 통해 Git 작업할 것이라고 알린다.

1. 로컬 저장소 생성
git init

1-1. 원격 저장소 연결 (처음 설정할 때만)
git remote add origin https://github.com/jpminlak/AI_bracket2

** 이 때 숨겨진 폴더 .git이 생성된다. git의 설정이므로 나중에 이걸 덮어쓰지 않도록 한다.


2. 브랜치 확인
git branch

2-1. 브랜치 생성 (처음 생성할 때만)
git branch main
(main 대신 각자의 브랜치 이름을 넣어서 할 수도 있다.)

* 브랜치 생성이 안되면 아래 [주석] 참조
오류 메시지: fatal: not a valid object name: 'master'

2-2. 브랜치 이동 (생성, 이동 둘 다 안될 때는 git branch -b main 으로 생성과 동시에 이동하도록 해보자)
git checkout main


3. 원격저장소의 파일 끝어오기	
git pull origin(원격 저장소 별칭) main

* fatal: refusing to merge unrelated histories 오류가 발생할 경우 이렇게 해보자
git pull origin [브랜치 이름] --allow-unrelated-histories


4. 추가할 파일을 붙여넣는다. 이 때 폴더 통째로 하위 폴더에 그냥 넣는다. 기존 폴더에 덮어쓰지 말자.


5. 추가 파일 대기시키기 (이상한 오류 메시지가 뜨면 다시 한 번 git add . 를 실행해보자! 보통 git add. 명령을 2번 해야 진짜 add . 명령이 말을 듣는다.)
git add .

5-1. git add . 명령이 제대로 실행됐는지 상태 확인 (익숙해지만 생략 가능)
git status

6. 커밋 하기
git commit -m "커밋 내용"

7. 푸시
git push -u origin main

7-1. 강제 푸시 (푸시가 안되면 쓰자!)
git push -f origin master(현재 브랜치)

ls -al

git remote -v

* error: failed to push some refs to 오류가 뜰 때
> 원인 - 어떤 파일이 원격 저장소에는 있지만, 내 컴퓨터에 없는 경우 발생.
먼저 push를 하여 원격저장소의 파일을 내 컴퓨터로 가져온 뒤
add, commit, push 하기

파일 생성
touch README.md

파일 삭제
rm README.md
